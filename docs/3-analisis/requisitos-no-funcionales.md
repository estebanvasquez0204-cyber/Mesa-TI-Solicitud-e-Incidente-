# Requisitos No Funcionales

---

**RNF-001**
| Campo | Descripción |
|-------|-------------|
| **ID** | RNF-001 |
| **Categoría** | Facilidad de uso |
| **Descripción** | El sistema debe poder ser usado por una persona que no sabe de sistemas. |
| **Métrica** | La persona debe poder registrar un caso en máximo 3 pantallas claras y sin pedir ayuda a otra persona. |

---

**RNF-002**
| Campo | Descripción |
|-------|-------------|
| **ID** | RNF-002 |
| **Categoría** | Claridad |
| **Descripción** | El sistema debe usar palabras sencillas en las pantallas principales. |
| **Métrica** | Todas las preguntas del registro deben entenderse en una prueba con usuarios no técnicos con mínimo 90 por ciento de respuestas correctas. |

---

**RNF-003**
| Campo | Descripción |
|-------|-------------|
| **ID** | RNF-003 |
| **Categoría** | Claridad |
| **Descripción** | El sistema debe mostrar una explicación visible en cada pregunta que pueda generar dudas. |
| **Métrica** | Cada pregunta importante debe tener una frase de ayuda o ejemplo visible sin que el usuario tenga que buscarlo. |

---

**RNF-004**
| Campo | Descripción |
|-------|-------------|
| **ID** | RNF-004 |
| **Categoría** | Uso en diferentes dispositivos |
| **Descripción** | El sistema debe funcionar en computador y celular. |
| **Métrica** | El usuario debe poder registrar, consultar y revisar sus casos desde ambos dispositivos sin perder información. |

---

**RNF-005**
| Campo | Descripción |
|-------|-------------|
| **ID** | RNF-005 |
| **Categoría** | Rapidez |
| **Descripción** | El sistema debe responder rápido en las acciones más usadas. |
| **Métrica** | Consultar un caso debe tomar menos de 3 segundos y registrar un caso debe tomar menos de 5 segundos. |

---

**RNF-006**
| Campo | Descripción |
|-------|-------------|
| **ID** | RNF-006 |
| **Categoría** | Protección de información |
| **Descripción** | El sistema debe proteger la información de cada usuario. |
| **Métrica** | Cada usuario solo debe poder ver sus propios casos, excepto el personal autorizado. |

---

**RNF-007**
| Campo | Descripción |
|-------|-------------|
| **ID** | RNF-007 |
| **Categoría** | Protección del historial |
| **Descripción** | El registro de cambios del caso no debe poder alterarse. |
| **Métrica** | Los cambios guardados no deben poder editarse ni borrarse por ningún usuario, incluyendo administrador, personal de soporte y usuarios solicitantes. |

---

**RNF-008**
| Campo | Descripción |
|-------|-------------|
| **ID** | RNF-008 |
| **Categoría** | Disponibilidad |
| **Descripción** | El sistema debe estar disponible durante la jornada laboral. |
| **Métrica** | El sistema debe estar disponible al menos el 99.5 por ciento del tiempo laboral. |

---

**RNF-009**
| Campo | Descripción |
|-------|-------------|
| **ID** | RNF-009 |
| **Categoría** | Avisos oportunos |
| **Descripción** | Los avisos sobre cambios del caso deben llegar rápidamente. |
| **Métrica** | El aviso debe enviarse en menos de 30 segundos después del cambio. |

---

**RNF-010**
| Campo | Descripción |
|-------|-------------|
| **ID** | RNF-010 |
| **Categoría** | Facilidad para soporte |
| **Descripción** | El personal de soporte debe poder reconocer rápidamente los casos más importantes. |
| **Métrica** | Los casos de mayor importancia deben verse desde la pantalla principal sin más de un clic. |

---

**RNF-011**
| Campo | Descripción |
|-------|-------------|
| **ID** | RNF-011 |
| **Categoría** | Conservación de información |
| **Descripción** | El sistema debe guardar los casos cerrados para consultas futuras. |
| **Métrica** | Los casos cerrados deben conservarse mínimo 5 años. |

---

**RNF-012**
| Campo | Descripción |
|-------|-------------|
| **ID** | RNF-012 |
| **Categoría** | Respaldo de información |
| **Descripción** | El sistema debe reducir el riesgo de pérdida de información. |
| **Métrica** | El sistema debe guardar borradores cada 30 segundos mientras el usuario registra un caso. |

---

**RNF-013**
| Campo | Descripción |
|-------|-------------|
| **ID** | RNF-013 |
| **Categoría** | Facilidad de seguimiento |
| **Descripción** | La situación actual del caso debe ser fácil de encontrar. |
| **Métrica** | El usuario debe poder ver cómo va su caso en máximo 2 clics. |

---

**RNF-014**
| Campo | Descripción |
|-------|-------------|
| **ID** | RNF-014 |
| **Categoría** | Crecimiento |
| **Descripción** | El sistema debe permitir que aumente la cantidad de usuarios sin afectar las funciones principales. |
| **Métrica** | El sistema debe soportar pasar de 200 a 500 usuarios registrados sin afectar el registro y consulta de casos. |
