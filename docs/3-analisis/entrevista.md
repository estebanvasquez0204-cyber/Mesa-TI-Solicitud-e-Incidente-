# Entrevista - Resultados por Pregunta

| Pregunta | Hallazgos principales |
|----------|----------------------|
| P1. Rol del usuario | La mayoría de participantes actúan como usuarios que solicitan ayuda, aunque también hay personal de TI y roles de coordinación. |
| P2. Claridad para solicitar ayuda | Algunos usuarios tienen claro cómo pedir ayuda, pero muchos no tienen claridad total sobre el proceso. |
| P3. Confirmación de solicitud | En varios casos los usuarios no saben si su solicitud fue recibida o atendida. |
| P4. Tiempo de respuesta | Se evidencia que en muchos casos las respuestas se demoran más de lo esperado. |
| P5. Uso de múltiples canales | La mayoría considera que el uso de varios medios (WhatsApp, correo, verbal) genera desorden. |
| P6. Calidad de la información | Algunos usuarios envían información completa, pero muchos deben complementar después. |
| P7. Canal más usado | El medio más utilizado para solicitar ayuda es WhatsApp, seguido del correo electrónico. |
| P8. Acción ante falta de respuesta | Los usuarios suelen volver a escribir, usar otro medio o preguntar a otra persona de TI. |
| P9. Canal ideal | La mayoría prefiere un único sistema o formulario centralizado para registrar las solicitudes. |
| P10. Dificultades actuales | Se destacan problemas como demoras, falta de claridad, mala comunicación y dificultad para saber el estado del caso. |
| P11. Tipo de solicitud | Los usuarios reportan tanto problemas urgentes como solicitudes de nuevos servicios. |
| P12. Clasificación de solicitudes | Muchos prefieren describir el problema y que el sistema o el área de TI lo clasifique. |
| P13. Tiempo esperado de respuesta | La mayoría espera respuestas rápidas, desde menos de una hora hasta el mismo día. |

## Conclusión

Los resultados muestran que actualmente la empresa presenta problemas importantes en la gestión de solicitudes de TI, debido al uso de múltiples canales, la falta de seguimiento y la poca claridad en la atención.

La mayoría de los usuarios requiere un sistema único donde puedan registrar sus solicitudes, recibir confirmación y conocer el estado de atención en todo momento.

Esto justifica la necesidad de implementar una Mesa de Ayuda TI que permita mejorar la organización, la comunicación y la rapidez en la atención de los incidentes.
